## 🎯 Metodología de Aprendizaje

### Principio 70-20-10
- **70% Práctica:** Ejercicios hands-on, proyectos reales
- **20% Mentoría/Colaboración:** Comunidades, foros, revisión de código
- **10% Teoría:** Documentación, videos, cursos

### Uso de IA como Acelerador (Recomendado)

La IA puede reducir 30-40% el tiempo perdido en debugging. Úsala con propósito:

**Cuando estés bloqueado en un error:**
```
"Estoy aprendiendo Power Platform. Tengo este error en Power Automate:
[pegar el mensaje de error completo]
El flujo hace: [descripción breve]
¿Cuál es la causa y cómo lo corrijo?"
```

**Para debuggear fórmulas Power Fx:**
```
"Esta fórmula Power Fx devuelve Blank pero debería devolver [valor esperado]:
[pegar fórmula]
El datasource es Dataverse, tabla [nombre], y la Gallery tiene estos registros: [ejemplo]"
```

**Para generar datos de prueba rápido:**
```
"Genera 15 registros JSON de prueba en español para una tabla Dataverse con estos campos:
Título (text), Categoría (choice: Hardware/Software/Red), Prioridad (choice: Baja/Media/Alta/Crítica),
Estado (choice: Nueva/En Proceso/Resuelta), Fecha Solicitud (últimos 60 días)"
```

**Para entender un concepto DAX:**
```
"Explícame en términos simples por qué esta measure DAX devuelve resultados diferentes
al filtrar por año vs. al ver el total: [pegar measure]"
```

**Regla importante:** Usa IA para entender el error y la solución, no para copiar código sin entenderlo. Practica reescribir la solución con tus palabras después.

## ⚙️ Configuración de Entorno de Desarrollo

### Requisitos Mínimos
- **PC/Laptop:** Windows 10/11, 8GB RAM (16GB recomendado)
- **Navegador:** Edge/Chrome actualizado
- **Conectividad:** Internet estable
- **Cuenta Microsoft:** Personal o corporativa

### Setup Inicial (Paso a Paso)

#### 1. Cuenta de Desarrollador Microsoft 365
```
1. Ve a: https://developer.microsoft.com/microsoft-365/dev-program
2. Clic en "Join now"
3. Inicia sesión con cuenta Microsoft
4. Completa perfil (selecciona "Learning and exploration")
5. Espera provisión del tenant (5-10 min)
6. Recibirás: nombreusuario@xxxxxxx.onmicrosoft.com
7. Guarda credenciales de forma segura
```

#### 2. Entorno Power Platform
```
1. Ve a: https://admin.powerplatform.microsoft.com
2. Inicia sesión con tu cuenta de desarrollador
3. Verás tu entorno por defecto
4. Crea entornos adicionales:
   - DEV (desarrollo)
   - QA (pruebas)
   - PROD (producción simulada)
```

#### 3. Dynamics 365 Trial
```
1. Ve a: https://trials.dynamics.com
2. Solicita trial de Customer Service o Sales
3. Usa tu cuenta de desarrollador M365
4. Trial de 30 días (renovable cada mes: repite pasos 1-3)
   Alternativa sin vencimiento: Power Platform Admin Center → Recursos →
   Aplicaciones Dynamics 365 → instalar Customer Service desde tu tenant E5
```

#### 4. Power BI Desktop
```
1. Descarga: https://aka.ms/pbidesktop
2. Instala en Windows
3. Inicia sesión con tu cuenta M365
```

#### 5. Visual Studio Code + Extensiones
```
1. Descarga VS Code: https://code.visualstudio.com
2. Instala extensiones:
   - Power Platform Tools
   - Power Platform Admin View
   - Azure DevOps (opcional)
```

#### 6. Power Platform CLI
```
PowerShell (como administrador):
1. winget install Microsoft.PowerPlatformCLI
   (o descarga desde: https://aka.ms/PowerPlatformCLI)
2. Verifica: pac --version
```

#### 7. XrmToolBox (Nivel 2+)
```
1. Descarga: https://www.xrmtoolbox.com
2. Extrae ZIP
3. Ejecuta XrmToolBox.exe
4. Instala plugins según necesites
```

## 🎓 Filosofía de Certificaciones

### ¿Cuándo Certificarse?
- **PL-900:** Después de completar Nivel 1 (fundamentos sólidos)
- **PL-200:** A mitad/final de Nivel 2 (ya trabajas soluciones)
- **PL-400:** Final de Nivel 3 (desarrollo profesional)
- **PL-600:** Final de Nivel 4 (arquitectura empresarial)

### ¿Son Necesarias?
- **Sí para:** Validación externa, credibilidad profesional, roles enterprise
- **No son suficientes:** Certificación sin experiencia práctica = valor limitado
- **Balance ideal:** Práctica intensa + certificación como validación
