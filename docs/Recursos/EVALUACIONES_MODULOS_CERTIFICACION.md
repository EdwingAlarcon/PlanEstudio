# 📝 Banco de Preguntas por Módulo (Formato Microsoft Learn / Certificación)

> Si quieres experiencia tipo examen (navegación, selección, resultado y reintento), usa el **Simulador de Evaluaciones** en Recursos.

Estas evaluaciones están diseñadas con estructura inspirada en evaluaciones oficiales de Microsoft Learn y exámenes de certificación (sin replicar preguntas oficiales).

## Estructura recomendada por módulo

- 8 ítems por módulo
- 5 preguntas de selección única
- 1 pregunta de selección múltiple
- 1 ítem de ordenamiento / correspondencia
- 1 caso práctico corto basado en escenario
- Umbral sugerido de aprobación: **70%** (con caso práctico obligatorio)

---

## Nivel 1 — Básico (Módulos 1–8)

### Módulo 1: Introducción al Ecosistema Power Platform
1. **Selección única:** ¿Qué componente es el almacén de datos nativo relacional de Power Platform?
2. **Selección múltiple:** Selecciona dos responsabilidades del Power Platform Admin Center.
3. **Caso práctico:** Una empresa usa correos manuales para solicitudes internas. Propón qué usarías de Power Apps, Dataverse, Power Automate y Power BI, justificando cada elección.

### Módulo 2: Dataverse - Fundamentos y Modelado Básico
1. **Selección única:** ¿Qué tipo de relación usarías para “un solicitante tiene muchas solicitudes”?
2. **Selección única:** ¿Cuál es la ventaja principal de usar prefijo de publisher en columnas personalizadas?
3. **Caso práctico:** Diseña un modelo mínimo para “Mesa de ayuda TI” con al menos 2 tablas, 1 relación y 1 regla de negocio.

### Módulo 3: Power Apps Canvas - Primeras Aplicaciones
1. **Selección única:** ¿Qué propiedad se usa normalmente para ejecutar inicialización al abrir la app?
2. **Selección múltiple:** Elige dos controles adecuados para listar y buscar registros.
3. **Caso práctico:** Crea el flujo lógico de una app Canvas para registrar solicitudes con validación de campos obligatorios.

### Módulo 4: Power Apps Model-Driven - Apps Basadas en Datos
1. **Selección única:** ¿Qué artefacto define cómo se visualizan listas filtradas de registros?
2. **Selección única:** ¿Qué opción aplica lógica sin código directamente en formularios/tablas?
3. **Caso práctico:** Diseña una app Model-Driven para seguimiento de casos con vistas por estado y formulario de detalle.

### Módulo 5: Power Automate - Automatización Básica
1. **Selección única:** ¿Qué elemento dispara la ejecución de un flujo?
2. **Selección única:** ¿Qué acción usarías para enviar notificación al crear un registro?
3. **Caso práctico:** Define un flujo de aprobación básico para solicitudes con estados “Nueva”, “Aprobada”, “Rechazada”.

### Módulo 6: Power BI - Reportes y Dashboards Básicos
1. **Selección única:** ¿Qué modelo de datos se recomienda para analítica escalable en BI?
2. **Selección múltiple:** Selecciona dos visuales útiles para seguimiento de KPI operativos.
3. **Caso práctico:** Diseña un dashboard para tickets TI con métricas de volumen, prioridad y tiempo promedio de resolución.

### Módulo 7: Fundamentos de Power Fx y Expresiones
1. **Selección única:** ¿Qué función usarías para buscar un solo registro por clave?
2. **Selección única:** ¿Qué función usarías para actualizar/crear registros en Dataverse desde Canvas?
3. **Caso práctico:** Escribe la lógica Power Fx para impedir guardar si prioridad está vacía y mostrar mensaje al usuario.

### Módulo 8: Primer Proyecto Integrado
1. **Selección única:** ¿Cuál es el objetivo principal de una solución integrada en Power Platform?
2. **Ordenamiento:** Ordena: modelado de datos, app de captura, automatización, analítica.
3. **Caso práctico:** Diseña una solución extremo a extremo para gestión de incidencias internas.

## Rúbrica común de calificación (aplicable a todos los módulos)

| Criterio | Peso |
|---|---:|
| Precisión técnica en preguntas objetivas | 40% |
| Calidad de decisión en escenarios | 30% |
| Diseño/ejecución de caso práctico | 30% |

### Escala sugerida
- **90–100%**: Dominio listo para aplicar en proyecto real.
- **70–89%**: Competencia funcional; requiere refuerzo puntual.
- **<70%**: Reforzar módulo antes de avanzar al siguiente.
